1. 프로그램 개요

ClassMate는 학교별 명단 파일을 읽어 등록작업파일과 안내 파일을 만들고, 필요 시 전체 명단 파일에 작업 결과까지 기록하는 프로그램입니다.

1-1. 두 가지 모드

모드용도
반편성 모드신입생·전입생·전출생·교직원 명단을 받아 등록작업파일과 ID/PW 안내 파일을 자동 생성합니다.
명단비교 모드학교에서 받은 재학생 명단과 서버 명부를 비교하여 이미 아이디가 있는 학생과 없는 학생을 분류합니다. 중복 아이디 생성을 방지하고 전입·전출 후보를 추출합니다.

1-2. 공통 작업 흐름

단계설명
1. 초기 설정작업 폴더, 전체 명단 파일, 열 매핑 확인 후 작업 시작
2. 학교 선택검색창에 학교명 입력 후 적용
3. 스캔파일 구조 자동 분석, 시작행 확인 및 조정, 경고·오류 확인
4. 실행파일 생성, 완료 확인, 보류 항목 처리
5. 명단 반영전체 명단 파일에 작업 결과 기록 (선택)

1-3. 알림 레벨

레벨의미
오류즉시 해결해야 합니다. 오류가 있으면 다음 단계로 진행할 수 없습니다.
경고실행은 가능하지만 결과를 주의 깊게 확인해야 합니다.
보류자동 처리가 불가한 항목입니다. 담당자가 직접 확인하고 수동으로 처리해야 합니다.
정보참고용 안내입니다. 별도 조치가 필요하지 않습니다.

2. 초기 설정

초기 설정에서는 작업 폴더와 전체 명단 파일을 연결하고, 열 매핑을 한 번 설정합니다.

2-1. 준비 파일

기능설명
학교 검색검색창 자동완성 소스. 파일에 없는 학교는 검색되지 않습니다.
학교 폴더 매칭전체 명단의 학교명과 작업 폴더 안의 폴더명을 대조하여 매칭합니다.
도메인 조회학교별 홈페이지 도메인을 읽어 안내문 및 ID 생성에 활용합니다.
작업 결과 기록실행 완료 후 종류별 처리 완료 여부, 이메일 도착일·발송일, 담당자명을 해당 학교 행에 기록합니다.
전체 명단 파일 주의사항
  • .xlsx 형식만 지원합니다. .xlsb는 지원하지 않습니다.
  • 명단 반영 시 파일이 열려 있으면 저장에 실패합니다. 공유 폴더에 있는 경우 다른 컴퓨터에서 사용 중인지도 확인하세요.

2-2. 작업 폴더 구조

프로그램이 파일을 자동으로 찾으려면 아래 구조를 지켜야 합니다. 작업 폴더를 지정하면 resources/ 관련 폴더가 자동으로 생성됩니다.

위치설명
작업폴더/resources/이름에 resources가 포함된 폴더가 1개 있어야 합니다.
작업폴더/resources/templates/등록 템플릿과 안내 템플릿 .xlsx 파일을 둡니다.
작업폴더/resources/notices/안내문 .txt 파일을 둡니다.
작업폴더/resources/DB/학교전체명단.xlsb 파일을 둡니다.
작업폴더/○○초등학교/학교별 작업 폴더입니다.
학교 폴더 안신입생 명단.xlsx, 전입생 명단.xlsx, 전출생 명단.xlsx, 교직원 명단.xlsx, 필요 시 학생명부.xlsx를 둡니다.
작업폴더/○○초등학교/작업/실행 후 결과 파일이 저장되는 폴더입니다.
작업폴더/○○초등학교/작업/_backup/기존 결과 파일을 자동 백업하는 폴더입니다.
학교 폴더 매칭 규칙

학교 폴더는 전체 명단 파일의 학교명을 기준으로 찾습니다. 학교명과 완전히 같게 맞추는 방식이 가장 안전합니다. 학교명이 포함된 폴더도 인식할 수 있지만, 같은 학교명이 들어간 폴더가 2개 이상이면 오류가 발생합니다.

2-3. 파일 형식 공통 규칙

  • 모든 명단 파일은 .xlsx 형식만 지원합니다.
  • 같은 종류의 입력 파일(신입생·전입생·전출생·교직원)이 2개 이상이면 오류가 발생합니다.
  • 시트가 2개 이상인 파일은 첫 번째 시트만 읽습니다. 경고가 표시됩니다.
  • 파일이 열려 있는 상태에서는 읽기 또는 저장이 실패할 수 있습니다.

3. 반편성 모드 사용법

반편성 모드 사용 순서만 먼저 따라가고, 오류나 경고는 필요할 때 부록 1에서 확인하세요.

3-1. 학교 선택

좌측 사이드바 검색창에 학교명을 입력하면 전체 명단 파일에서 일치하는 학교를 검색하여 목록으로 표시합니다. 선택 후 [학교 적용] 버튼을 눌러 학교를 확정합니다.

  • 학교를 선택하면 사이드바에 마지막 작업 이력(작업일, 담당자, 처리 건수)이 표시됩니다.
  • 안내문 내 홈페이지 주소는 초기 설정에서 지정한 전체 명단 파일의 도메인 열에서 자동으로 가져옵니다.
  • [새 학교 시작] 버튼을 누르면 현재 스캔·실행 결과가 초기화됩니다. 아직 명단에 반영하지 않은 작업이 있으면 확인 팝업이 표시됩니다.

3-2. 학교 구분 자동 판정

학교명의 마지막 글자를 기준으로 학교 구분과 최대 학년을 자동으로 판정합니다.

3-3. 스캔

스캔 탭에서 [파일 내용 스캔] 버튼을 누르면 학교 폴더 안의 파일을 자동으로 분석합니다.

신입생

  • 신입생 파일에 2학년 이상이 포함된 경우, 아이디 규칙 확인을 위해 학생명부가 필요합니다.

전입생

  • 전입생 파일이 있으면 기존 아이디 규칙 참고를 위해 학생명부가 반드시 필요합니다.
  • 전입생의 ID는 명부에서 같은 학년 학생들의 입학년도 최빈값을 기준으로 생성됩니다.
  • 명부에 이미 같은 학생이 존재하면 보류 처리됩니다.

전출생

  • 관리용ID 신청 건이 존재하지 않는 경우 경고가 표시됩니다. 신청 열이 올바르게 인식됐는지 확인하세요.
  • 신청 여부는 값이 입력되어 있으면 신청, 비어 있으면 미신청으로 처리됩니다.
개학일의 두 가지 역할
  1. 학년도 판단: 개학일 기준으로 명부 학년도를 결정합니다. 작업 시점이 개학일 이전이면 작년 명부로 판단하고 학년에 +1 보정을 적용합니다.
  2. 퇴원일자 결정: 작업 시점이 개학일 이전이면 퇴원일자를 개학일로 설정합니다. 개학일 이후 작업하면 퇴원일자를 작업일로 설정합니다.

교직원

  • 학습용ID, 관리용ID 신청 열이 없으면 실행 시 오류가 발생합니다.
  • 직위 열은 선택 사항입니다. 입력값이 있으면 안내 파일에 포함됩니다.

3-6. 명부 기준일 팝업

학생명부 파일의 마지막 수정일(다운로드 시점)과 작업일이 다를 때 팝업이 표시됩니다.

선택지의미
수정일 사용명부 파일의 마지막 수정일을 기준으로 학년도를 판정합니다.
작업일 사용오늘 작업일을 기준으로 학년도를 판정합니다.

기준일에 따라 학년도 판정이 달라집니다.

  • 작업일 < 개학일 → 작년 학년도
  • 작업일 > 개학일 → 올해 학년도

3-7. 학년도 ID 규칙

학생 ID는 입학년도 4자리 + 학생 이름으로 구성됩니다.

상황입학년도 결정 방식
명부가 있는 경우명부의 같은 학년 학생들 중 입학년도 최빈값을 사용합니다.
예) 명부 3학년 15명 중 12명의 ID가 2022로 시작 → 3학년 전입생 입학년도 = 2022
명부는 있으나 해당 학년 데이터 없음인접 학년의 명부값을 기준으로 추정합니다.
예) 4학년=2021이 확인되면 3학년=2022, 5학년=2020으로 추정

6. 명단비교 모드 사용법

6-1. 개요

학교에서 받은 재학생 명단과 서버 학생명부를 비교하여 이미 아이디가 있는 학생과 없는 학생을 자동으로 분류합니다. 중복 아이디 생성을 방지하고 전입·전출 후보를 추려내는 것이 목적입니다.

프로그램 상단의 모드 전환 버튼으로 전환합니다. 비교 모드에서 생성된 결과 파일을 반편성 모드 작업에 활용할 수 있습니다.

6-2. 재학생 명단 파일 규칙

  • 파일명에 재학생 또는 신청자 키워드가 포함되어야 자동 인식됩니다.
  • 파일 형식은 .xlsx만 지원합니다.
  • 시트가 여러 개이면 첫 번째 시트만 읽습니다.
  • 학년, 반, 이름 열이 존재해야 합니다. 인식 키워드는 반편성 모드의 학생 명단 규칙과 같습니다.

  • 위치: resources/notices/
  • 형식: .txt 파일 (UTF-8 인코딩)

{school_name}선택한 학교 이름서울가락초등학교
{year}현재 학년도 또는 올해 연도2026
{prev_year}이전 연도2025
{month}안내문에 넣을 월3
{day}안내문에 넣을 일4
{domain}전체 명단 파일에 등록된 학교 홈페이지 주소 또는 도메인school.readinggate.com

부록 4. FAQ

파일을 올바르게 넣었는데도 찾을 수 없다고 나옵니다

확장자 및 파일 형식을 확인하거나, 파일명에 필수 키워드가 정확히 포함되어 있는지 확인하세요. 예: 신입생명단.xlsx (O) / 학생목록.xlsx (X)

스캔 후 시작행 숫자가 잘못 감지되었습니다

파일명을 클릭하면 미리보기가 열립니다. 실제 데이터 시작 행을 확인한 후 +/- 버튼으로 조정하거나 올바른 행 숫자를 직접 입력하세요.

체크박스를 선택해도 실행 버튼이 활성화되지 않습니다

감지된 파일 종류 모두에 체크가 필요합니다. 체크되지 않은 항목이 있는지 확인하세요. 스캔 자체가 완료되지 않은 경우에도 실행 버튼이 비활성화됩니다.

이미 작업한 학교를 다시 작업하면 기존 파일은 사라지나요?

같은 파일이 있으면 덮어쓰지 않고 _backup 폴더에 날짜·시간이 붙어 자동 보관됩니다.

학교를 검색하면 일치하는 검색 결과가 없습니다

초기 설정에서 연결한 전체 명단 내 학교열에 정확히 일치하는 이름이 존재해야 합니다. 띄어쓰기, 괄호 등 세부 표기까지 맞춰 주세요. 폴더명은 학교명 기준으로 맞추는 것을 권장합니다.

명단 반영 후 이메일 도착일·발송일이 기록되지 않습니다

사이드바에서 해당 항목 왼쪽 체크박스를 선택한 경우에만 기록됩니다. 체크 후 다시 명단 반영 버튼을 눌러 주세요.

비고 입력란에 작성한 내용은 어디에 저장되나요?

실행이 성공적으로 완료된 경우에만 학교 폴더 안에 작업메모_YYMMDD_HHMM.txt 파일로 저장됩니다. 전체 명단 파일에는 기록되지 않습니다.
