## 1장. 프로그램 개요

### 두 가지 모드

ClassMate는 상단 탭에서 두 가지 모드를 선택할 수 있습니다.

| 모드 | 용도 |
|---|---|
| 반편성 모드 | 학교에서 신입생·전입생·전출생·교직원 명단을 받았을 때 등록작업파일과 ID/PW 안내 파일을 자동 생성합니다. |
| 명단비교 모드 | 학교에서 재학생 명단 또는 신규 신청자 명단을 받았을 때 서버 명부와 비교하여 이미 아이디가 있는 학생과 없는 학생을 분류합니다. 중복 아이디 생성 방지가 목적입니다. |

### 공통 작업 흐름

두 모드 모두 스캔 → 실행 2단계로 진행됩니다.

```
1단계. 초기 설정   작업 폴더, 전체 명단 파일, 열 매핑 확인 후 작업 시작
2단계. 학교 선택   검색창에 학교명 입력 후 적용
3단계. 스캔        파일 구조 자동 분석, 시작행 확인 및 조정, 경고·오류 확인
4단계. 실행        파일 생성, 완료 확인, 보류 항목 처리
5단계. 명단 반영   전체 명단 파일에 작업 결과 기록 (선택)
```

### 알림 레벨

프로그램의 모든 알림은 다음 네 가지 수준으로 표시됩니다.

| 레벨 | 의미 |
|---|---|
| 오류 | 즉시 해결해야 합니다. 오류가 있으면 다음 단계로 진행할 수 없습니다. |
| 경고 | 실행은 가능하지만 결과를 주의 깊게 확인해야 합니다. |
| 보류 | 자동 처리가 불가한 항목입니다. 담당자가 직접 확인하고 수동으로 처리해야 합니다. |
| 정보 | 참고용 안내입니다. 별도 조치가 필요하지 않습니다. |

## 2장. 공통 준비사항

### 2-1. 전체 명단 파일

전체 명단 파일은 프로그램의 루트 데이터입니다. 학교 검색, 작업 결과 기록, 도메인 조회 등 핵심 기능이 모두 이 파일을 기반으로 동작합니다.

**역할**

| 기능 | 설명 |
|---|---|
| 학교 검색 | 검색창 자동완성 소스. 파일에 없는 학교는 검색되지 않습니다. |
| 학교 폴더 매칭 | 명단의 학교명과 작업 폴더 안의 폴더명을 대조하여 매칭합니다. |
| 도메인 조회 | 학교별 홈페이지 도메인을 읽어 ID 생성에 활용합니다. |
| 작업 결과 기록 | 실행 완료 후 각 종류별 처리 완료 여부, 이메일 도착일·발송일, 담당자명을 해당 학교 행에 기록합니다. |

**파일 형식 주의사항**
- `.xlsx` 형식만 지원합니다. `.xlsb` 파일은 지원하지 않습니다.
- 작업 결과를 반영하기 전 파일이 열려 있으면 저장이 실패합니다. 반드시 닫은 상태에서 반영하세요.
- 전체 명단의 학교명과 작업 폴더 안의 폴더명이 정확히 일치해야 합니다. 띄어쓰기 포함.

### 2-2. 작업 폴더 구조

프로그램이 파일을 자동으로 찾으려면 아래 구조를 반드시 지켜야 합니다.

```
작업폴더/
├── resources/              이름에 "resources" 포함 필수 (반드시 1개)
│   ├── templates/          등록·안내 템플릿 xlsx 각 1개
│   ├── notices/            안내문 텍스트 파일 (.txt)
│   └── DB/                 학교전체명단 xlsb 파일
├── ○○초등학교/             학교 폴더 (학교명 그대로)
│   ├── 신입생 명단.xlsx
│   ├── 전입생 명단.xlsx
│   ├── 전출생 명단.xlsx
│   ├── 교직원 명단.xlsx
│   └── 학생명부.xlsx       전입·전출 있을 때 필수
└── △△중학교/
```

### 2-3. 파일 형식 규칙

- 모든 입력 파일은 `.xlsx` 형식만 지원합니다. `.xls` 구형 형식은 오류가 납니다. Excel에서 다른 이름으로 저장 → Excel 통합 문서(.xlsx)로 변환하세요.
- 같은 종류의 파일이 2개 이상이면 오류가 납니다. 하나만 남겨 주세요.
- 파일이 열려 있는 상태에서는 읽기가 실패할 수 있습니다. (`~$파일명.xlsx` 생성 여부 확인)
- 시트가 2개 이상인 파일은 첫 번째 시트만 읽습니다.

### 2-4. resources 폴더 규칙

- `resources` 이름을 포함한 폴더는 반드시 1개여야 합니다. 2개 이상이면 오류가 납니다.
- `templates/` 안에 파일명에 "등록" 포함된 xlsx → 정확히 1개
- `templates/` 안에 파일명에 "안내" 포함된 xlsx → 정확히 1개
- `notices/` 안의 `.txt` 파일 이름이 안내문 선택 목록에 그대로 표시됩니다. 파일을 추가하면 목록에 자동으로 나타나고, 삭제하면 자동으로 사라집니다.

## 3장. 초기 설정

프로그램을 처음 실행하거나 설정을 변경할 때 초기 설정 화면이 표시됩니다.

### 3-1. 입력 항목

| 항목 | 설명 |
|---|---|
| 작업 폴더 | 학교별 폴더와 resources 폴더가 있는 상위 폴더. 찾아보기로 선택하거나 직접 입력. |
| 학교 전체 명단 파일 | 학교 목록이 담긴 xlsx 파일. 선택 후 열 매핑 설정이 필요합니다. |
| 개학일 | 학년도 판단 및 전출생 퇴원일자 결정에 사용됩니다. |
| 작업일 | 기본값은 오늘 날짜입니다. |
| 작업자 | 담당자 이름. 작업 이력 및 전체 명단 기록에 사용됩니다. |

### 3-2. 열 매핑 설정

전체 명단 파일을 선택하면 열 매핑 설정 다이얼로그가 열립니다. 프로그램에 파일의 열 구조를 알려주는 과정입니다. 열 번호는 A=1, B=2, C=3 순입니다.

| 항목 | 설명 |
|---|---|
| 시트 이름 | 학교 목록이 있는 시트명 |
| 헤더 행 | 열 제목이 있는 행 번호 |
| 데이터 시작 행 | 실제 학교 데이터가 시작하는 행 번호 |
| 학교명 열 | 학교 이름이 있는 열 번호 |
| 도메인 열 | 학교 홈페이지 도메인 열 (없으면 0) |
| 이메일 도착일 열 | 학교에서 이메일 받은 날짜 기록 열 |
| 이메일 발송일 열 | 완료 이메일 보낸 날짜 기록 열 |
| 담당자 열 | 작업자 이름 기록 열 |
| 신입생 ~ 교직원 열 | 종류별 처리 결과 기록 열 |
| 자료실 순번 열 | 자료실 순번 기록 열 (없으면 0) |

### 3-3. 기본 설정 저장·불러오기

- **기본 설정 저장**: 현재 입력된 작업 폴더, 명단 파일 경로, 열 매핑 등을 저장합니다. 다음 실행 시 자동으로 불러옵니다.
- **기본 설정 불러오기**: 저장된 설정을 현재 화면에 불러옵니다.

## 4장. 반편성 모드 — 학교 선택 및 스캔

### 4-1. 학교 선택

작업 시작 후 좌측 사이드바 검색창에 학교명을 입력하면 전체 명단 파일에서 일치하는 학교를 검색하여 목록으로 표시합니다. 선택 후 **적용** 버튼을 눌러 학교를 확정합니다.

학교를 선택하면 사이드바에 해당 학교의 마지막 작업 이력(작업일, 담당자, 처리 건수)이 표시됩니다.

**새 학교 시작**: 다른 학교로 변경할 때 사이드바 상단의 새 학교 시작 버튼을 누릅니다. 현재 스캔·실행 결과가 초기화됩니다. 전체 명단에 아직 반영하지 않은 작업이 있으면 확인 팝업이 표시됩니다.

### 4-2. 학교 구분 자동 판정

학교명의 마지막 글자를 기준으로 학교 구분을 자동으로 판정합니다.

| 학교명 끝 글자 | 판정 결과 |
|---|---|
| 초 | 초등부 (1~6학년) |
| 중 | 중등부 (1~3학년) |
| 고 | 고등부 (1~3학년) |
| 초중 으로 끝남 | 초중 혼합 (1~9학년) |
| 그 외 | 자동 판정 불가 |

자동 판정이 불가한 경우 **학교 구분 선택 팝업**이 표시됩니다. 초등부 / 중등부 / 고등부 / 기타 중 해당하는 구분을 선택한 후 다시 스캔을 진행합니다.

### 4-3. 스캔

스캔 탭에서 **파일 내용 스캔** 버튼을 누르면 학교 폴더 안의 파일을 자동으로 분석합니다.

**스캔 결과 표**

스캔이 완료되면 파일별로 아래 정보가 표시됩니다.

| 항목 | 설명 |
|---|---|
| 파일명 | 감지된 파일 이름. 클릭하면 미리보기가 열립니다. 더블클릭하면 실제 파일이 열립니다. |
| 시작행 | 데이터가 시작하는 행 번호. 자동 감지된 값이 표시됩니다. |
| 체크박스 | 시작행을 확인했음을 표시합니다. 실행 버튼 활성화의 조건입니다. |
| 경고 아이콘 | 파일 구조에 경고가 있으면 표시됩니다. 클릭하면 미리보기에서 해당 행이 노란색으로 강조됩니다. |

**시작행 조정**

자동 감지된 시작행이 잘못된 경우 스캔 결과 표에서 +/- 버튼으로 직접 조정합니다. 파일명을 클릭하여 미리보기를 열면 데이터가 어느 행부터 시작하는지 확인할 수 있습니다.

**체크박스와 실행 버튼 활성화**

파일이 감지된 종류(신입생, 전입생, 전출생, 교직원)에 대해 모든 체크박스를 선택해야 실행 버튼이 활성화됩니다. 체크박스는 "이 파일의 시작행을 확인했습니다"라는 확인 행위입니다. 파일이 없는 종류는 체크박스가 비활성화되며 체크할 필요가 없습니다.

### 4-4. 스캔 단계의 입력 파일별 규칙

#### 신입생

파일명에 "신입생" 또는 "신입"이 포함되어야 합니다.

| 열 | 필수 여부 | 인식하는 열 제목 |
|---|---|---|
| 이름 | 필수 | 성명, 이름, 학생이름 |
| 학년 | 필수 | 학년 |
| 반 | 필수 | 반, 학급 |
| 출석번호 | 선택 | 번호, 번 |

신입생 파일에 1학년 외 학년이 포함된 경우, 학생명부가 없으면 보류 메시지가 표시됩니다. 학교 폴더에 명부를 추가하거나 사이드바에서 해당 학년의 ID prefix 연도를 직접 입력한 후 재실행할 수 있습니다.

#### 전입생

파일명에 "전입생" 또는 "전입"이 포함되어야 합니다.

| 열 | 필수 여부 | 인식하는 열 제목 |
|---|---|---|
| 이름 | 필수 | 성명, 이름, 학생이름 |
| 학년 | 필수 | 학년 |
| 반 | 필수 | 반, 학급 |
| 출석번호 | 선택 | 번호, 번, 출석번호 |
| 비고 | 선택 | 비고, 메모, 특이사항 |

전입생 파일이 있으면 학생명부 파일이 반드시 있어야 합니다. 파일명에 "학생명부"가 포함되어야 하며, 여러 개 있으면 가장 최근에 수정된 파일을 사용합니다.

#### 전출생

파일명에 "전출생" 또는 "전출"이 포함되어야 합니다.

| 열 | 필수 여부 | 인식하는 열 제목 |
|---|---|---|
| 이름 | 필수 | 성명, 이름 |
| 학년 | 필수 | 학년 |
| 반 | 필수 | 반, 학급 |
| 비고 | 선택 | 비고, 메모, 특이사항 |

전출생 파일이 있으면 학생명부와 개학일이 모두 필요합니다.

**개학일의 역할**
- 학년도 판단: 개학일 기준으로 명부가 올해 것인지 작년 것인지 결정합니다. 개학 전이면 명부 학년에 +1 보정을 적용합니다.
- 퇴원일자 결정: 개학 전 작업이면 퇴원일자를 개학일로 맞춥니다. 개학 이후 작업이면 작업일 당일을 퇴원일자로 사용합니다.

#### 교직원

파일명에 "교사", "교원", "교직원" 중 하나가 포함되어야 합니다.

| 열 | 필수 여부 | 인식하는 열 제목 |
|---|---|---|
| 이름 | 필수 | 성명, 이름, 성함, 교사명, 교사이름, 교사성명, 교원명, 교원이름, 교원성명, 교직원명, 교직원이름, 교직원성명, 선생님이름, 선생님성명, 선생님명, 담당자명, 담당자이름 |
| 학습용 ID 신청 | 필수 | 학습용id신청, 학습용id, 학습용, 학습용아이디 |
| 관리용 ID 신청 | 필수 | 관리용id신청, 관리용id, 관리용, 관리용아이디 |
| 직위 | 선택 | 직위, 담당, 직위담당, 직책, 직급 |

신청 열이 비어 있으면 미신청, 어떤 값이든 입력되어 있으면 신청으로 처리됩니다. 관리용 ID 신청자가 한 건도 없으면 스캔 단계에서 경고가 표시됩니다.

#### 학생 명부

파일명에 "학생명부"가 포함되어야 합니다.

명부 기준일은 파일의 마지막 수정일을 기준으로 자동 판정됩니다. 명부의 마지막 수정일과 작업일이 다른 경우 **명부 기준일 설정 팝업**이 표시됩니다.

| 선택지 | 의미 |
|---|---|
| 수정일 사용 | 명부 파일의 마지막 수정일을 기준으로 학년도를 판정합니다. |
| 작업일 사용 | 오늘 작업일을 기준으로 학년도를 판정합니다. |

명부 기준일에 따라 학년도 판정이 달라지므로 실제 상황에 맞는 날짜를 선택해야 합니다.

## 5장. 반편성 모드 — 실행 및 결과 확인

### 5-1. 실행

스캔이 완료되고 모든 체크박스를 선택하면 실행 탭의 **작업 실행** 버튼이 활성화됩니다.

실행 탭에서 확인할 수 있는 항목:

| 항목 | 설명 |
|---|---|
| 비고 입력란 | 이번 작업 관련 메모를 남길 수 있습니다. 입력 내용은 해당 학년도 작업 이력 파일(`work_history_YYYY.json`)에 저장됩니다. |
| 작업 실행 버튼 | 클릭하면 실행이 시작됩니다. 실행 중에는 버튼이 비활성화됩니다. |

### 5-2. 결과 파일

작업이 완료되면 학교 폴더 안에 `작업/` 폴더가 생성되고 파일이 저장됩니다.

| 파일명 | 내용 |
|---|---|
| `★{학교명}_등록작업파일(작업용).xlsx` | BM 등록용 파일 |
| `☆{학교명}_{종류}_ID,PW안내.xlsx` | 학생·교직원 안내용 파일 |

같은 파일이 이미 있으면 `작업/_backup/` 폴더에 날짜·시간이 붙어 자동 백업됩니다.

결과 파일 목록에서 파일명을 클릭하면 해당 파일이 바로 열립니다.

### 5-3. 결과 미리보기 및 필터

실행이 완료되면 실행 탭 하단에 결과 미리보기 표가 표시됩니다.

**시트 탭**: 등록작업파일의 시트별로 전환하여 내용을 확인할 수 있습니다.

**동명이인 필터**: 같은 학년 내 동명이인이 발생한 경우 상단에 "동명이인 N건" 알림이 표시됩니다. **동명이인만 보기** 버튼을 누르면 해당 행만 필터링하여 표시합니다. 동명이인 행은 미리보기에서 별도 색상으로 구분됩니다.

**검색**: 검색창에 이름 등을 입력하면 해당 내용이 포함된 행만 표시됩니다.

**행 색상 구분**

| 색상 | 의미 |
|---|---|
| 노란색 | 동명이인 구분자(A, B, C)가 부여된 행 |
| 분홍색 | 보류 처리된 행 |
| 회색 | 자동 제외된 행 |

### 5-4. 전입생 처리 상세

**ID 결정 방식**

전입생의 ID는 `입학년도 4자리 + 학교코드 + 순번`으로 구성됩니다. 프로그램은 학생 명부의 같은 학년 학생들 중 입학년도 최빈값을 전입생의 입학년도로 추정합니다.

예) 명부의 3학년 학생 15명 중 12명의 ID가 2022로 시작하면 → 전입생 3학년의 ID prefix = 2022

명부에 해당 학년 데이터가 없어 최빈값 산출이 불가하면 보류 처리됩니다. 이 경우 사이드바에서 해당 학년의 연도를 직접 입력 후 재실행합니다.

**동명이인 구분자 부여**

같은 학년 내에 같은 이름의 학생이 있으면 이름 뒤에 영문 대문자(A, B, C...)를 붙여 구분합니다.

| 상황 | 처리 방식 | 예시 |
|---|---|---|
| 명부에 이미 같은 이름이 있고 전입생 1명 추가 | 명부의 마지막 구분자 다음 번호 부여 | 명부: 김지우, 김지우A → 전입생: 김지우B |
| 명부에 없고 전입생끼리 같은 이름 2명 | A부터 순서대로 부여 | 전입생: 김지우, 김지우 → 김지우A, 김지우B |
| 신입생 파일 내 동명이인 | A부터 순서대로 부여 | 이수현, 이수현 → 이수현A, 이수현B |
| 중복 없음 | 구분자 없이 그대로 | 홍길동 → 홍길동 |

구분자가 붙은 이름은 등록작업파일에서 노란색으로 하이라이트 표시됩니다. 안내 파일에도 구분자가 포함된 이름 그대로 출력됩니다.

### 5-5. 보류 처리 이해

보류란 프로그램이 자동으로 판단하기 어려운 케이스를 담당자에게 알리는 상태입니다. 보류 항목은 자동 처리에서 제외되고 등록작업파일의 별도 시트에 사유와 함께 기록됩니다.

실행 완료 후 보류 항목이 있으면 실행 탭 상단에 보류 사유 목록이 표시됩니다.

**전입생 보류 케이스**

| 사유 | 발생 조건 | 담당자 조치 |
|---|---|---|
| 신입생 명단과 학년/반/이름이 동일합니다. | 전입생과 신입생이 동일인으로 의심됨 | 어느 쪽이 맞는지 확인 후 파일 수정 및 재실행 |
| 학생명부에 이미 존재하는 학생입니다. | 전입생이 명부에 같은 학년·반·이름으로 이미 있음 | 서버에 이미 계정이 있는 학생. 중복 등록 여부 확인 |
| 학생명부에 동일인으로 의심되는 학생이 있습니다. | 명부에 같은 학년·이름이 있으나 반이 다름 | 동일인 여부 직접 확인 후 수동 처리 |
| ID 규칙을 확인할 수 없어 확인이 필요합니다. | 명부에 해당 학년 데이터 없음 | 사이드바에서 해당 학년 ID 연도 직접 입력 후 재실행 |

**전출생 보류 케이스**

| 사유 | 발생 조건 | 담당자 조치 |
|---|---|---|
| 학생명부에 동명이인이 있어 확인이 필요합니다. | 전출생 이름이 명부에서 구분자로 여러 명 존재 | 명부에서 어느 학생인지 직접 확인 후 수동 처리 |
| 학생명부에서 후보가 여러 건 확인되어 확인이 필요합니다. | 같은 학년에 동명이인이 다른 반에 여러 명 있음 | 실제 전출생이 어느 반인지 확인 후 수동 처리 |
| 학생명부에서 중복 매칭되어 확인이 필요합니다. | 반+이름 조합으로 명부에서 여러 건 매칭됨 | 명부 데이터 확인 후 수동 처리 |
| 학생명부에서 확인되지 않아 확인이 필요합니다. | 명부에 없는 전출생 — 자동 판단 불가 | 직접 확인 후 수동 처리 |

**자동 제외**

명부에 존재하지 않는 전출생은 서버 미등록 또는 학년 불일치 등으로 추정되어 자동 제외됩니다. 자동 제외 항목은 등록작업파일의 퇴원 시트에 "자동 제외되었습니다" 사유로 기록됩니다.

## 6장. 명단비교 모드

### 6-1. 개요

학교에서 받은 명단과 서버의 학생 명부를 비교하여 이미 아이디가 있는 학생과 없는 학생을 자동으로 분류합니다. 중복 아이디 생성을 방지하고 신규 등록이 필요한 학생만 추려내는 것이 목적입니다.

비교 대상 학년: 2~6학년 (1학년은 신입생이므로 자동 제외)

### 6-2. 준비물

| 파일 | 파일명 키워드 | 비고 |
|---|---|---|
| 비교용 명단 | "재학생" 또는 "신청자" | 전체 명단도, 신청자 명단만도 사용 가능 |
| 학생 명부 | "학생명부" | 학교 폴더 안에 위치 |

**비교용 명단 필수 헤더**

| 열 | 필수 여부 | 인식하는 열 제목 |
|---|---|---|
| 학년 | 필수 | 학년 |
| 반 | 필수 | 반, 학급 |
| 이름 | 필수 | 이름, 성명, 학생이름 |

### 6-3. 명부 기준일 설정 팝업

명부 파일의 마지막 수정일과 작업일이 다른 경우 명부 기준일 설정 팝업이 표시됩니다.

| 선택지 | 의미 |
|---|---|
| 수정일 사용 | 명부 파일의 마지막 수정일을 기준으로 학년도를 판정합니다. |
| 작업일 사용 | 오늘 작업일을 기준으로 학년도를 판정합니다. |

### 6-4. 비교 결과 분류

| 분류 | 의미 | 담당자 조치 |
|---|---|---|
| 공통 학생 | 명단과 명부 양쪽에 모두 있음. 이미 아이디 있는 재학생. | 별도 조치 불필요 |
| 명부에만 있음 | 명단에 없음. 전출 가능성. | 전출생 명단 받아 반편성 모드로 처리 |
| 명단에만 있음 | 명부에 없음. 신규 등록 필요. | 반편성 모드에서 전입생 또는 신입생으로 처리 |
| 자동 판정 불가 | 이름·학년·반이 애매하게 겹침. | 담당자가 직접 확인 필요 |

### 6-5. 결과 파일

| 파일명 | 내용 |
|---|---|
| `{학교명}_명단비교 결과.xlsx` | 전체 비교 결과 (분류별 시트 포함) |

## 7장. 전체 명단 반영

실행 완료 후 사이드바 하단 **명단 반영** 버튼을 누르면 전체 명단 파일에 작업 결과가 기록됩니다.

### 7-1. 이메일 도착일 · 발송일 기록

사이드바의 메일 기록 항목에서 도착일·발송일 왼쪽 체크박스를 선택한 경우에만 해당 날짜가 기록됩니다. 체크하지 않으면 날짜가 기록되지 않습니다. 기본값은 모두 작업일로 설정됩니다.

### 7-2. 반영 완료 후 동작

- **명단 반영** 버튼은 반영 완료 후 **반영 완료** 표시로 바뀌며 비활성화됩니다.
- 새 학교 작업을 시작하면 버튼이 초기화됩니다.
- **파일 열기** 버튼은 전체 명단 파일 경로가 설정된 경우 언제든 사용할 수 있습니다.

### 7-3. 작업 이력 자동 기록

실행이 완료되면 명단 반영과 별개로 프로그램 내부 작업 이력(`work_history_YYYY.json`)에 아래 정보가 자동으로 저장됩니다.

| 항목 | 내용 |
|---|---|
| 작업일 | 실행 시점의 작업일 |
| 담당자 | 초기 설정의 작업자 이름 |
| 처리 건수 | 신입생·전입생·전출생·교직원 각 처리 건수 |
| 비고 | 실행 탭의 비고 입력란에 작성한 내용 |

이 이력은 다음에 같은 학교를 선택했을 때 사이드바에 "마지막 작업" 정보로 표시됩니다.

## 8장. 안내문 탭

안내문 탭에서는 학생·교직원에게 전달할 안내 문구를 작성하고 복사할 수 있습니다.

- `notices/` 폴더 안의 `.txt` 파일 목록이 선택 목록으로 표시됩니다.
- 안내문을 선택하면 내용이 표시됩니다.
- **복사** 버튼을 누르면 내용이 클립보드에 복사됩니다. 버튼이 잠깐 "복사됨" 표시로 바뀐 후 원래대로 돌아옵니다.

## 9장. 템플릿·안내문 수정 가이드

담당자가 바뀌거나 양식 변경이 필요할 때 참고하세요.

### 9-1. 안내문 (.txt) 추가·수정·삭제

코드를 건드릴 필요가 없습니다.

- 위치: `resources/notices/` 폴더
- 형식: `.txt` 파일 (UTF-8 인코딩)
- 파일명이 그대로 프로그램 안내문 선택 목록에 표시됩니다.
  - 예) `2026_전입생안내문.txt` → 목록에 "2026_전입생안내문" 으로 표시
- 파일 추가 → 목록에 자동으로 나타납니다.
- 파일 삭제 → 목록에서 자동으로 사라집니다.

### 9-2. 등록 템플릿 수정

**절대 변경 금지 — 변경 시 실행 오류 발생**

시트명 (정확히 아래 3개여야 합니다)
- `학생자료` / `직원정보` / `그룹반정보`

[학생자료] 시트 1행 열 제목
- `No`, `학생이름`, `ID`, `학교구분`, `학교`, `학년`, `수강반`

[직원정보] 시트 1행 열 제목
- `No`, `이름`, `아이디`, `권한부여`

[그룹반정보] 시트 1행 열 제목
- `그룹명`, `반명`, `수강료`, `담임명`, `FullMode`

**자유롭게 수정 가능한 항목**

- 셀 색상, 폰트, 열 너비, 테두리 등 서식 전체
- 열 순서 변경 (코드가 열 이름으로 찾으므로 위치가 바뀌어도 무관)
- 위 필수 열 외에 추가 열 삽입

### 9-3. 안내 템플릿 수정

**절대 변경 금지**
- 시트가 최소 1개 이상 있어야 합니다. 시트 전체 삭제 금지.
- 데이터는 4행부터 시작합니다. 1~3행은 코드가 건드리지 않으므로 자유롭게 수정 가능합니다.

**자유롭게 수정 가능한 항목**
- 시트 이름 변경
- 1~3행의 안내 문구, 로고, 서식
- 4행 이하 열 서식, 색상, 폰트, 열 너비

### 9-4. 템플릿 파일 교체 시 주의사항

1. 새 파일명에 반드시 "등록" 또는 "안내" 키워드가 포함되어야 합니다.
2. `resources/templates/` 폴더 안에 같은 종류 파일이 반드시 1개만 있어야 합니다.
3. 교체 후 반드시 스캔을 다시 실행하여 오류가 없는지 확인하세요.

## 10장. 알림 메시지 참고

### 10-1. 공통 오류

| 메시지 | 원인 | 해결 방법 |
|---|---|---|
| resources 폴더에 필요한 자료가 모두 포함되었는지 확인하세요. | templates 또는 notices 폴더 누락, 템플릿 파일 없음 | `resources/templates/`에 등록·안내 템플릿 각 1개 확인 |
| 작업 폴더 내에 resources 폴더가 여러 개 있습니다. | "resources"가 이름에 포함된 폴더가 2개 이상 | 하나만 남기고 나머지 이름 변경 또는 삭제 |
| templates 폴더에 등록 템플릿이 여러 개 있습니다. | "등록" 포함 xlsx가 2개 이상 | 하나만 남기고 나머지 삭제 또는 이름 변경 |
| templates 폴더에 안내 템플릿이 여러 개 있습니다. | "안내" 포함 xlsx가 2개 이상 | 하나만 남기고 나머지 삭제 또는 이름 변경 |
| 명단 파일은 .xlsx 형식이어야 합니다. | 전체 명단 파일이 .xls 구형 형식 | Excel에서 .xlsx로 다시 저장 후 재선택 |
| ○○학교를 명단 파일에서 찾을 수 없습니다. | 입력한 학교명이 전체 명단에 없음 | 전체 명단 파일의 학교명과 입력값 일치 여부 확인 |
| ○○학교 폴더를 찾을 수 없습니다. | 작업 폴더 안에 해당 학교 이름의 폴더가 없음 | 폴더명이 학교명과 정확히 일치하는지 확인 (띄어쓰기 포함) |
| ○○학교 폴더 후보가 여러 개입니다. | 유사한 이름의 폴더가 2개 이상 | 폴더 이름을 정확하게 하나만 유지 |
| 학교 폴더 내 명단 파일을 하나도 찾을 수 없습니다. | 신입생·전입생·전출생·교직원 파일 모두 없음 | 파일명 규칙 확인 후 파일 추가 |

### 10-2. 파일 구조 오류

| 메시지 | 원인 | 해결 방법 |
|---|---|---|
| .xls 형식은 지원하지 않습니다. | 파일이 구형 형식 | .xlsx로 저장 후 교체 |
| ○○ 명단 파일이 2개 이상 감지되었습니다. | 파일 중복 | 하나만 남기고 나머지 삭제 |
| ○○ 파일에서 헤더를 찾을 수 없습니다. | 열 제목 행 인식 불가 | 스캔 화면에서 시작행 조정 |
| ○○ 파일에서 ○○ 열을 찾을 수 없습니다. | 필수 열 없거나 열 제목이 다름 | 헤더 규칙 확인 후 파일 수정 |
| ○○ 파일에서 데이터 행을 찾을 수 없습니다. | 데이터가 0건 | 파일 내용 확인 |
| ○○ 파일에서 학생 명부가 필요합니다. | 명부 파일 없음 | 학교 폴더에 "학생명부" 포함된 xlsx 추가 후 재스캔 |
| 전출 처리에 필요한 개학일이 입력되지 않았습니다. | 개학일 미입력 | 사이드바에서 개학일 입력 후 다시 실행 |

### 10-3. 경고

| 메시지 | 의미 및 조치 |
|---|---|
| ○○ N행 '학년' 열 — 학년+반이 합쳐진 것 같습니다. | "3-2"처럼 합쳐진 값. 학년/반 열 분리 요청 권장. |
| ○○ N행 '학년' 열 — '학년' 글자 포함. | "3학년"처럼 글자 포함. 숫자만 입력되도록 수정 권장. |
| ○○ N행 '반' 열 — '반' 글자 포함. | "2반"처럼 글자 포함. 숫자만 입력되도록 수정 권장. |
| ○○ N행 '이름' 열 — 이름이 너무 짧습니다. | 1자 이하. 파일 확인. |
| ○○ N행이 비어 있습니다. | 데이터 중간 빈 행 존재. 빈 행 삭제 권장. |
| ○○ 데이터 영역에 병합된 셀이 있습니다. | 병합 셀 존재. 병합 해제 후 재시도 권장. |
| ○○ 시트가 두 개 이상입니다. 첫 번째 시트만 사용합니다. | 다중 시트. 데이터 시트가 첫 번째인지 확인. |
| ○○ 파일에 유치부 학생이 포함되어 있습니다. | 유치부 학생 자동 제외됨. 확인 후 진행. |
| 교직원 명단에 관리용 아이디 신청자가 한 건도 없습니다. | 관리용 ID 신청 열이 모두 비어 있음. 교직원 파일 확인. |
| 자동으로 학교를 판정할 수 없습니다. 학교 구분을 직접 선택해 주세요. | 학교명으로 초·중·고 판정 불가. 팝업에서 직접 선택. |
| 학생명부 기준일과 작업일이 다릅니다. | 명부 수정일과 작업일이 다름. 팝업에서 올바른 기준일 선택. |
| 전출 파일이 있습니다. 실행 전 개학일을 입력해 주세요. | 개학일 미입력 상태. 사이드바에서 입력 후 실행. |
| 동명이인 N건 — 실행 결과 탭 필터에서 확인해 주세요. | 동명이인 발생. 동명이인만 보기 버튼으로 확인. |

### 10-4. 팝업(모달) 발생 케이스

프로그램에서 팝업이 표시되는 경우는 다음과 같습니다.

| 팝업 | 발생 조건 | 처리 방법 |
|---|---|---|
| 학교 구분 선택 | 학교명으로 초·중·고 자동 판정이 불가한 경우 | 초등부 / 중등부 / 고등부 / 기타 중 선택 후 다시 스캔 |
| 명부 기준일 설정 | 명부 파일의 마지막 수정일과 작업일이 다른 경우 | 수정일 사용 또는 작업일 사용 중 선택 |
| 새 학교 시작 확인 | 새 학교 시작 버튼 클릭 시 학교가 선택된 상태 | 초기화 진행 또는 취소 |
| 미반영 경고 | 전체 명단에 반영하지 않은 상태에서 새 학교 시작 시 | 반영 후 진행 또는 미반영 상태로 진행 |
| 열 매핑 설정 | 전체 명단 파일 선택 후 (찾아보기 버튼 클릭 시) | 각 항목에 해당하는 열 번호 입력 후 설정 완료 |
| 스캔 로그 보기 | 스캔 탭에서 로그 보기 버튼 클릭 시 | 로그 확인 후 닫기 |

## 부록. 자주 묻는 질문

**Q. 파일을 올바르게 넣었는데도 찾을 수 없다고 나옵니다.**
파일명에 필수 키워드가 정확히 포함되어 있는지 확인하세요. 예) `신입생명단.xlsx` (O) / `학생목록.xlsx` (X)

**Q. 스캔 후 시작행 숫자가 잘못 감지되었습니다.**
스캔 화면에서 파일명을 클릭하면 미리보기가 열립니다. 실제 데이터 시작 행을 확인한 후 +/- 버튼으로 조정하세요.

**Q. 체크박스를 선택해도 실행 버튼이 활성화되지 않습니다.**
감지된 파일 종류 모두에 체크가 필요합니다. 체크되지 않은 항목이 있는지 확인하세요. 스캔 자체가 완료되지 않은 경우에도 실행 버튼이 비활성화됩니다.

**Q. 동명이인 구분자(A, B, C)가 안내 파일에도 그대로 나옵니다.**
구분자가 붙은 이름 그대로 안내 파일에 출력됩니다. 학생에게 전달할 때 별도 안내가 필요할 수 있습니다.

**Q. 명단비교 후 전입·전출 처리는 어디서 합니까?**
명단비교는 분류만 합니다. 실제 처리는 반편성 모드에서 해당 파일을 받아 진행해야 합니다.

**Q. 작업 결과가 기존 파일을 덮어씁니까?**
같은 파일이 있으면 덮어쓰지 않고 `_backup` 폴더에 날짜·시간이 붙어 자동 보관됩니다.

**Q. 전체 명단 파일의 학교명과 폴더명이 다릅니다.**
두 이름이 정확히 일치해야 합니다. 띄어쓰기, 괄호 등 세부 표기까지 맞춰 주세요. 전체 명단 파일의 학교명을 기준으로 폴더명을 맞추는 것을 권장합니다.

**Q. 명단 반영 후 이메일 도착일·발송일이 기록되지 않습니다.**
사이드바에서 해당 항목 왼쪽 체크박스를 선택한 경우에만 기록됩니다. 체크 후 다시 명단 반영 버튼을 눌러 주세요.

**Q. 비고 입력란에 작성한 내용은 어디에 저장됩니까?**
실행 탭의 비고 입력란 내용은 프로그램 내부 작업 이력 파일(`work_history_YYYY.json`)에 저장됩니다. 전체 명단 파일에는 기록되지 않습니다.
