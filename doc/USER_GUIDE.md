# RG_webview — 사용자 가이드

> 리딩게이트 반이동 자동화 도구 사용 안내서입니다.

---

## 시작 전 준비사항

### 필요한 것

1. **작업 폴더** — 학교별 폴더와 resources 폴더가 들어 있는 상위 폴더
2. **전체 명단 파일** — 학교 목록이 담긴 xlsx 파일 (사내 공유 폴더)
3. **입력 파일** — 학교에서 받은 신입생/전입생/전출생/교직원 명단 (.xlsx)

---

## 작업 폴더 구조

프로그램이 자동으로 파일을 찾기 위해 **아래 구조를 반드시 지켜야 합니다.**

```
작업폴더/
│
├── resources/              ← 이름에 'resources' 포함 (필수)
│   ├── templates/          ← 등록/안내 템플릿 파일
│   └── notices/            ← 안내문 텍스트 파일 (.txt)
│
├── ○○초등학교/             ← 학교 폴더 (학교명 그대로)
│   ├── 신입생 명단.xlsx
│   ├── 전입생 명단.xlsx
│   ├── 전출생 명단.xlsx
│   └── 교직원 명단.xlsx
│
├── △△중학교/
│   └── ...
│
└── ...
```

---

## 파일 이름 규칙

학교에서 받은 파일 이름에 **아래 키워드가 반드시 포함되어야** 합니다.

| 파일 종류 | 이름에 포함해야 할 키워드 |
|---|---|
| 신입생 명단 | `신입생` 또는 `신입` |
| 전입생 명단 | `전입생` 또는 `전입` |
| 전출생 명단 | `전출생` 또는 `전출` |
| 교직원 명단 | `교사`, `교원`, `교직원` 중 하나 |
| 학생 명부 | `학생명부` |
| 재학생 명렬표 (명단비교용) | `명렬표`, `명렬`, `재학생`, `학생명단` 중 하나 |

### 주의사항

- **파일 형식은 반드시 `.xlsx`** 입니다. `.xls`(구형)는 지원하지 않습니다.
  - Excel에서 `다른 이름으로 저장 → Excel 통합 문서(.xlsx)`로 변환하세요.
- 같은 종류의 파일이 **2개 이상** 있으면 오류가 납니다. 하나만 남겨 주세요.
- 파일이 열려 있는 상태에서는 사용할 수 없습니다. (열려 있으면 `~$파일명.xlsx`가 생성됩니다)

## 입력 파일 헤더 요건

프로그램이 자동으로 헤더 행을 찾습니다. 아래 조건을 맞춰 주세요.

### 신입생 / 전입생 / 전출생

다음 열이 있어야 합니다:

| 열 | 인식하는 이름 |
|---|---|
| 이름 **(필수)** | 성명, 이름, 학생이름 |
| 학년 | 학년 |
| 반 | 반, 학급 |

### 교직원

| 열 | 인식하는 이름 |
|---|---|
| 이름 **(필수)** | 성명, 이름, 성함, 교사명, 교원명, 교직원명, 선생님이름 등 |
| 학습용 아이디 신청 | 학습용id신청, 학습용id, 학습용, 학습용아이디 |
| 관리용 아이디 신청 | 관리용id신청, 관리용id, 관리용, 관리용아이디 |

> 신청 열에 값이 있는 행 = 신청한 것으로 처리합니다.

### 재학생 명렬표 (명단비교 탭)

| 열 | 인식하는 이름 |
|---|---|
| 학년 **(필수)** | 학년 |
| 반 **(필수)** | 반, 학급 |
| 이름 **(필수)** | 이름, 성명, 학생이름 |

> 명단비교는 **2~6학년** 기준으로 비교합니다. 1학년은 비교 대상에서 제외됩니다.

---

## 학생 명부 (전입/전출 있을 때)

전입생 또는 전출생 파일이 있으면 **학생 명부**가 필요합니다.

- 학생 명부 파일을 해당 **학교 폴더 안에** 넣어 주세요.
- 파일명에 반드시 `학생명부`가 포함되어야 합니다.
- 여러 개 있으면 가장 최근에 수정된 파일을 사용합니다.

---

## resources 폴더 구조

```
resources/
├── templates/
│   ├── *등록*.xlsx   ← 파일명에 '등록' 포함, 정확히 1개
│   └── *안내*.xlsx   ← 파일명에 '안내' 포함, 정확히 1개
└── notices/
    ├── 안내문A.txt   ← UTF-8 텍스트 파일
    └── 안내문B.txt
```

- templates 폴더에 등록 템플릿과 안내 템플릿이 각각 **정확히 1개씩** 있어야 합니다.
- notices 폴더의 txt 파일 이름이 안내문 탭에서 선택 목록으로 표시됩니다.

## 실행 결과 파일

작업이 완료되면 학교 폴더 안에 `작업/` 폴더가 생기고 파일이 저장됩니다.

| 파일 | 설명 |
|---|---|
| `★{학교명}_등록작업파일(작업용).xlsx` | BM 등록용 파일 |
| `☆{학교명}_{종류}_ID,PW안내.xlsx` | 학생/교직원 안내용 파일 |
| `{학교명}_명단비교 결과.xlsx` | 명단비교 탭 결과 |

> 같은 파일이 이미 있으면 `작업/_backup/` 폴더에 날짜·시간이 붙어 자동 백업됩니다.

## 자주 발생하는 오류

| 오류 메시지 | 원인 | 해결 방법 |
|---|---|---|
| 학교 폴더를 찾을 수 없습니다 | 학교명과 폴더명이 다름 | 전체 명단에 있는 학교명과 폴더명이 일치하는지 확인 |
| .xls 형식은 지원하지 않습니다 | 구형 엑셀 형식 | Excel에서 .xlsx로 다시 저장 |
| 명단 파일이 2개 이상 감지되었습니다 | 같은 종류 파일 중복 | 하나만 남기고 나머지 삭제 또는 이동 |
| 헤더를 찾을 수 없습니다 | 열 제목 행을 못 찾음 | 파일을 열어 헤더 행 위치 확인, 스캔 화면에서 시작행 조정 |
| 학생명부가 필요합니다 | 전입/전출 있는데 명부 없음 | 학교 폴더에 '학생명부' 포함된 xlsx 파일 추가 |
| resources 폴더에 필요한 자료가 없습니다 | 템플릿 파일 없거나 이름 오류 | resources/templates/ 폴더 확인 |
| 교직원 관리용 아이디 신청이 한 건도 없습니다 | 신청 열이 모두 비어 있음 | 교직원 파일에서 관리용id 신청 열 확인 |

## 작업 순서

```
1. 초기 설정
   └── 작업 폴더 선택 → 전체 명단 파일 선택 → 열 매핑 확인 → 시작

2. 학교 선택
   └── 검색창에 학교명 입력 → 적용

3. 스캔 (파일 확인)
   └── 스캔 시작 → 파일별 시작행 확인 → 오류/경고 확인

4. 실행 (파일 생성)
   └── 실행 시작 → 완료 확인 → 출력 파일 열기

5. 전체 명단 반영 (선택)
   └── 명단에 기록 → 이메일 발송 후 발송일 기록
```

## 전체 명단 열 매핑

전체 명단 파일(xlsx)의 열 위치를 프로그램에 알려줘야 합니다.  
초기 설정 화면에서 `열 매핑 설정`을 클릭하면 대화상자가 열립니다.

설정해야 하는 항목:

| 항목 | 설명 |
|---|---|
| 시트 이름 | 학교 목록이 있는 시트명 |
| 헤더 행 | 열 제목이 있는 행 번호 |
| 데이터 시작 행 | 실제 학교 데이터가 시작하는 행 번호 |
| 학교명 열 | 학교 이름이 있는 열 번호 |
| 홈페이지 열 | 도메인 정보 열 (없으면 0) |
| 이메일 도착일 열 | 학교에서 이메일 받은 날짜 기록 열 |
| 이메일 발송일 열 | 완료 이메일 보낸 날짜 기록 열 |
| 담당자 열 | 작업자 이름 기록 열 |
| 신입생~교직원 열 | 종류별 처리 결과 기록 열 |

> 열 번호는 A=1, B=2, C=3 ... 순서입니다.

## 명단비교 탭 사용법

재학생 명렬표와 학생 명부를 비교해 전입/전출 대상을 자동 분류합니다.

1. 상단에서 `명단비교` 모드 선택
2. 학교 선택
3. 재학생 명렬표 파일을 학교 폴더에 넣기
4. 스캔 → 결과 확인 → 실행
5. `{학교명}_명단비교 결과.xlsx` 확인

**비교 기준 학년:** 2~6학년 (1학년은 신입생이므로 제외)

결과 구분:
- **공통 학생**: 명렬표와 명부 양쪽에 있음 (정상 재학생)
- **명부에만 있음**: 명렬표에 없음 → 전출 가능성
- **명렬표에만 있음**: 명부에 없음 → 전입 가능성
- **자동 판정 불가**: 이름/학년/반이 애매하게 겹침 → 수동 확인 필요
