ClassMate 사용자 메뉴얼

이 문서는 ClassMate 프로그램을 사용하는 담당자를 위한 메뉴얼입니다. 양식 수정·템플릿 교체 등 운영 관리 내용은 관리자 가이드 섹션에 별도로 정리했습니다.

1장. 프로그램 개요

ClassMate는 리딩게이트 새학기 반이동 작업을 자동화하는 사내 도구입니다. 학교에서 받은 명단 파일을 입력받아 BM 등록용 파일과 ID/PW 안내 파일을 생성합니다.

두 가지 모드

상단 탭에서 두 가지 모드 중 하나를 선택하여 작업합니다.

새학기 반편성
학교에서 신입생·전입생·전출생·교직원 명단을 받았을 때 등록작업파일과 ID/PW 안내 파일을 자동 생성합니다.
재학생 명단 비교
학교에서 재학생 명단 또는 신규 신청자 명단을 받았을 때 서버 명부와 비교하여 이미 아이디가 있는 학생과 없는 학생을 분류합니다. 중복 아이디 생성 방지가 목적입니다.

공통 작업 흐름

기본 설정 → 학교 선택 → 스캔 → 실행 / 결과 → (선택) 명단 반영 · 안내문 확인

알림 레벨

프로그램의 모든 알림은 다음 네 가지 수준으로 표시됩니다.

레벨의미
오류즉시 해결해야 합니다. 오류가 있으면 다음 단계로 진행할 수 없습니다.
경고실행은 가능하지만 결과를 주의 깊게 확인해야 합니다.
보류자동 처리가 불가한 항목입니다. 담당자가 직접 확인하고 수동으로 처리해야 합니다.
정보참고용 안내입니다. 별도 조치가 필요하지 않습니다.

2장. 시작 전 준비사항

프로그램이 정상 동작하려면 아래 조건을 모두 갖춰야 합니다. 조건이 맞지 않으면 스캔 또는 실행 단계에서 오류가 발생합니다.

전체 명단 파일

전체 명단 파일은 프로그램의 루트 데이터입니다. 학교 검색, 작업 결과 기록, 도메인 조회 등 핵심 기능이 모두 이 파일을 기반으로 동작합니다.

기능설명
학교 검색검색창 자동완성 소스입니다. 파일에 없는 학교는 검색되지 않습니다.
학교 폴더 매칭명단의 학교명과 작업 폴더 안의 폴더명을 대조하여 매칭합니다.
도메인 조회학교별 홈페이지 도메인을 읽어 ID 생성에 활용합니다.
작업 결과 기록실행 완료 후 각 종류별 처리 완료 여부, 이메일 도착일·발송일, 담당자명을 해당 학교 행에 기록합니다.
주의 파일 형식은 .xlsx만 지원합니다. .xlsb는 지원하지 않습니다. 작업 결과를 반영하기 전 파일이 열려 있으면 저장이 실패합니다. 전체 명단의 학교명과 작업 폴더의 폴더명은 띄어쓰기까지 정확히 일치해야 합니다.

작업 폴더 구조

프로그램이 파일을 자동으로 찾으려면 아래 구조를 반드시 지켜야 합니다.

작업폴더/
├── resources/              이름에 "resources" 포함 필수 (반드시 1개)
│   ├── templates/          등록·안내 템플릿 xlsx 각 1개
│   ├── notices/            안내문 텍스트 파일 (.txt)
│   └── DB/                 학교전체명단 xlsb 파일
├── ○○초등학교/             학교 폴더 (학교명 그대로)
│   ├── 신입생 명단.xlsx
│   ├── 전입생 명단.xlsx
│   ├── 전출생 명단.xlsx
│   ├── 교직원 명단.xlsx
│   └── 학생명부.xlsx       전입·전출 있을 때 필수
└── △△중학교/
주의 resources 이름을 포함한 폴더가 2개 이상이면 오류가 발생합니다. 반드시 1개만 유지하세요. templates/ 안에 "등록" 포함 xlsx와 "안내" 포함 xlsx가 각각 정확히 1개씩 있어야 합니다.

지원 파일 형식

  • 모든 입력 파일은 .xlsx 형식만 지원합니다. .xls 구형 형식은 오류가 발생합니다. Excel에서 다른 이름으로 저장 → Excel 통합 문서(.xlsx)로 변환하세요.
  • 같은 종류의 파일이 2개 이상이면 오류가 발생합니다. 하나만 남겨 주세요.
  • 파일이 열려 있는 상태에서는 읽기가 실패할 수 있습니다. (~$파일명.xlsx 파일이 있으면 열려 있는 상태)
  • 시트가 2개 이상인 파일은 첫 번째 시트만 읽습니다.

4장. 학교 선택

작업 시작 후 사이드바 검색창에 학교명을 입력하면 전체 명단 파일에서 일치하는 학교를 검색합니다. 선택 후 적용 버튼을 눌러 확정합니다. 학교를 선택하면 사이드바에 해당 학교의 마지막 작업 이력(작업일, 담당자, 처리 건수)이 표시됩니다.

학교 구분 자동 판정

학교명의 마지막 글자를 기준으로 학교 구분을 자동으로 판정합니다.

학교명 끝 글자판정 결과적용 학년
초초등부1~6학년
중중등부1~3학년
고고등부1~3학년
초중 으로 끝남초중 혼합1~9학년
그 외자동 판정 불가 → 팝업 표시—
자동 판정이 불가한 경우 스캔 중 학교 구분 선택 팝업이 표시됩니다. 초등부 / 중등부 / 고등부 / 기타 중 선택 후 다시 스캔을 진행합니다.

새 학교 시작

다른 학교로 변경할 때 단계 바 하단의 + 새 학교 시작 버튼을 누릅니다. 현재 스캔·실행 결과가 초기화됩니다. 전체 명단에 아직 반영하지 않은 작업이 있으면 확인 팝업이 표시됩니다.

5장. 스캔 검수

스캔 탭에서 파일 내용 스캔 버튼을 누르면 학교 폴더 안의 파일을 자동으로 분석합니다.

스캔 결과 표

항목설명
파일명감지된 파일 이름. 클릭하면 하단 미리보기가 열립니다. 더블클릭하면 실제 파일이 열립니다. 경고가 있으면 주황색으로 표시됩니다.
시작행데이터가 시작하는 행 번호. 자동 감지된 값이 표시됩니다. +/− 버튼으로 조정할 수 있습니다.
확인 (체크박스)시작행을 확인했음을 표시합니다. 파일이 감지된 모든 종류에 체크해야 실행 버튼이 활성화됩니다. 파일이 없는 종류는 체크박스가 비활성화됩니다.
실행 버튼이 활성화되지 않는 경우 감지된 파일 종류 모두에 체크박스가 선택되어야 합니다. 스캔 자체가 완료되지 않은 경우에도 실행 버튼이 비활성화됩니다.

미리보기 (선택 파일 전체 검수)

스캔 결과 표 아래의 미리보기 패널에서 파일 내용을 행 단위로 확인할 수 있습니다.

  • 펼치기 버튼으로 미리보기 영역을 열고 닫습니다.
  • 노란색 행: 경고가 있는 행 (학년/반/이름 형식 이상 등)
  • 의심 행만: 경고가 있는 행만 필터링합니다.
  • 동명이인만: 동명이인이 발생한 행만 필터링합니다.

학년도 아이디 규칙 (사이드바)

스캔 완료 후 사이드바의 학년도 아이디 규칙 섹션에 학년별 ID prefix(입학년도 앞 4자리)가 자동으로 채워집니다. 명부에서 추정한 값이며, 잘못된 경우 직접 수정 후 적용 버튼을 누릅니다.

6장. 실행 / 결과

스캔이 완료되고 모든 체크박스를 선택하면 실행 탭의 작업 실행 버튼이 활성화됩니다. 실행 탭의 비고 입력란에 이번 작업 관련 메모를 남길 수 있습니다. 입력 내용은 프로그램 내부 작업 이력 파일(work_history_YYYY.json)에 저장되며, 전체 명단 파일에는 기록되지 않습니다.

결과 파일

전입생 처리 상세

ID 결정 방식

전입생의 ID는 입학년도 4자리 + 학교코드 + 순번으로 구성됩니다. 프로그램은 학생 명부의 같은 학년 학생들 중 입학년도 최빈값을 전입생의 입학년도로 추정합니다.

예) 명부의 3학년 학생 15명 중 12명의 ID가 2022로 시작 → 전입생 3학년의 ID prefix = 2022

명부에 해당 학년 데이터가 없어 최빈값 산출이 불가하면 보류 처리됩니다. 이 경우 사이드바에서 해당 학년의 연도를 직접 입력 후 재실행합니다.

동명이인 구분자 부여

같은 학년 내에 같은 이름의 학생이 있으면 이름 뒤에 영문 대문자(A, B, C...)를 붙여 구분합니다.

상황처리 방식예시
명부에 이미 같은 이름이 있고 전입생 1명 추가명부의 마지막 구분자 다음 번호 부여명부: 김지우, 김지우A → 전입생: 김지우B
명부에 없고 전입생끼리 같은 이름 2명A부터 순서대로 부여전입생: 김지우, 김지우 → 김지우A, 김지우B
신입생 파일 내 동명이인A부터 순서대로 부여이수현, 이수현 → 이수현A, 이수현B
중복 없음구분자 없이 그대로홍길동 → 홍길동

구분자가 붙은 이름은 등록작업파일에서 노란색으로 하이라이트 표시됩니다. 안내 파일에도 구분자가 포함된 이름 그대로 출력됩니다.

보류 처리 이해

보류란 프로그램이 자동으로 판단하기 어려운 케이스를 담당자에게 알리는 상태입니다. 보류 항목은 자동 처리에서 제외되고 등록작업파일의 별도 시트(전입_보류, 퇴원_보류)에 사유와 함께 기록됩니다.

7장. 안내문

안내문 탭에서는 학생·교직원에게 전달할 안내 문구를 작성하고 복사할 수 있습니다. 실행 완료 후 학교명·학년도·개학일·홈페이지 주소가 안내문에 자동으로 반영됩니다.

  • notices/ 폴더 안의 .txt 파일 목록이 왼쪽 선택 목록으로 표시됩니다.
  • 복사 버튼을 누르면 내용이 클립보드에 복사됩니다. 버튼이 잠깐 "복사됨" 표시로 바뀐 후 원래대로 돌아옵니다.
  • 초기화 버튼을 누르면 자동 반영된 내용이 원본 템플릿으로 돌아갑니다.
  • 실행 탭의 → 버튼을 클릭하면 안내문 탭으로 이동합니다.

10장. 상황별 처리

실행 버튼이 활성화되지 않습니다.

  2. 스캔 결과 표에서 파일이 감지된 모든 종류의 체크박스가 선택되었는지 확인합니다.

전입생이 보류 처리되었습니다.

등록작업파일의 전입_보류 시트에서 사유를 확인합니다. 사유에 따라 다음과 같이 처리합니다.

  • 신입생 명단과 동일: 신입생인지 전입생인지 확인 후 해당 파일에서 삭제하고 재실행
  • 명부에 이미 존재: 서버에서 계정 존재 여부 확인 후 필요시 수동 처리
  • 동일인으로 의심: 명부의 반 정보 확인 후 동일인이면 수동 처리
  • ID 규칙 확인 불가: 사이드바에서 해당 학년 연도 직접 입력 후 재실행

안내문 추가·수정·삭제

안내문 파일은 코드를 건드리지 않고 자유롭게 관리할 수 있습니다.

  • 위치: resources/notices/ 폴더
  • 형식: .txt 파일 (UTF-8 인코딩)
  • 파일명이 그대로 프로그램 안내문 선택 목록에 표시됩니다.
예) 2026_전입생안내문.txt → 목록에 "2026_전입생안내문"으로 표시

파일을 추가하면 목록에 자동으로 나타나고, 삭제하면 자동으로 사라집니다. 별도 코드 수정이나 재시작이 필요하지 않습니다.

인코딩 주의 파일을 새로 작성할 때 반드시 UTF-8 인코딩으로 저장하세요. Windows 메모장에서는 저장 시 인코딩을 "UTF-8"로 선택합니다.

등록 템플릿 수정 범위

등록 템플릿(파일명에 "등록" 포함된 xlsx)은 코드가 시트명과 열 제목을 직접 참조합니다.

절대 변경 금지 — 변경 시 실행 오류 발생

시트명 (정확히 아래 3개여야 합니다)

  • 학생자료 / 직원정보 / 그룹반정보

[학생자료] 시트 1행 열 제목

No, 학생이름, ID, 학교구분, 학교, 학년, 수강반

[직원정보] 시트 1행 열 제목

No, 이름, 아이디, 권한부여

[그룹반정보] 시트 1행 열 제목

그룹명, 반명, 수강료, 담임명, FullMode

자유롭게 수정 가능한 항목

  • 셀 색상, 폰트, 열 너비, 테두리 등 서식 전체
  • 열 순서 변경 (코드가 열 이름으로 찾으므로 위치가 바뀌어도 무관)
  • 위 필수 열 외에 추가 열 삽입
  • 열 제목 외의 셀 내용

안내 템플릿 수정 범위

절대 변경 금지

  • 시트가 최소 1개 이상 있어야 합니다. 시트 전체 삭제 금지.
  • 데이터는 4행부터 시작합니다. 1~3행은 코드가 건드리지 않습니다.

자유롭게 수정 가능한 항목

  • 시트 이름 변경
  • 1~3행의 안내 문구, 로고, 서식
  • 4행 이하 열 서식, 색상, 폰트, 열 너비

템플릿 교체 시 점검사항

  1. 새 파일명에 반드시 "등록" 또는 "안내" 키워드가 포함되어야 합니다.
  2. resources/templates/ 폴더 안에 같은 종류 파일이 반드시 1개만 있어야 합니다. 2개 이상이면 오류가 발생합니다.
  3. 교체 후 반드시 프로그램에서 스캔을 실행하여 오류가 없는지 확인합니다.
  4. 등록 템플릿 교체 시 시트명과 필수 열 제목이 기존과 동일한지 확인합니다.
교체 전 백업 권장 기존 템플릿 파일을 다른 위치에 복사해 두고 교체 작업을 진행하세요.

자주 묻는 질문

파일을 올바르게 넣었는데도 찾을 수 없다고 나옵니다.

파일명에 필수 키워드가 정확히 포함되어 있는지 확인하세요.
예) 신입생명단.xlsx (O) / 학생목록.xlsx (X)

체크박스를 선택해도 실행 버튼이 활성화되지 않습니다.

감지된 파일 종류 모두에 체크가 필요합니다. 스캔 자체가 완료되지 않은 경우에도 실행 버튼이 비활성화됩니다.

동명이인 구분자(A, B, C)가 안내 파일에도 그대로 나옵니다.

구분자가 붙은 이름 그대로 안내 파일에 출력됩니다. 학생에게 전달할 때 별도 안내가 필요할 수 있습니다.

명단비교 후 전입·전출 처리는 어디서 합니까?

명단비교는 분류만 합니다. 실제 처리는 반편성 모드에서 해당 파일을 받아 진행해야 합니다.

작업 결과가 기존 파일을 덮어씁니까?

같은 파일이 있으면 덮어쓰지 않고 _backup 폴더에 날짜·시간이 붙어 자동 보관됩니다.

비고 입력란에 작성한 내용은 어디에 저장됩니까?

실행 탭의 비고 입력란 내용은 프로그램 내부 작업 이력 파일(work_history_YYYY.json)에 저장됩니다. 전체 명단 파일에는 기록되지 않습니다.

명단 반영 후 이메일 도착일·발송일이 기록되지 않습니다.

사이드바에서 해당 항목 왼쪽 체크박스를 선택한 경우에만 기록됩니다. 체크 후 다시 명단 반영 버튼을 눌러 주세요.

전체 명단 파일의 학교명과 폴더명이 다릅니다.

두 이름이 정확히 일치해야 합니다. 띄어쓰기, 괄호 등 세부 표기까지 맞춰 주세요.
